# v2.2 — Real pan/zoom canvas + compact controls

## Why (owner feedback, Turkish)

Owner tested v2.1 and is unhappy with the zoom: "Zoom in/out yapıyorum ama
'Senin bilgisayarın' ve 'GitHub' kısımları hâlâ aynı büyüklükte kalıyor…
zoom out yapmanın hiçbir faydası yok." He wants both panels to live on one
big canvas that genuinely zooms and pans (move left/right, grow/shrink), so
zoom is meaningful. He also finds the bottom button bar too bulky ("çok yer
kaplıyor… belki sol tarafa alabiliriz, daha kompakt"). He loves the command
view — keep it exactly. He wants a quick, not-overly-complex revision.

## Root cause (measured in-browser)

- Container is ~1248×436 (aspect 2.86); SVG viewBox is 1280×720 (aspect 1.78).
- `preserveAspectRatio` letterboxes the scene to fit by HEIGHT and centers it,
  so panels already use full height but only ~62% of width — huge side margins.
- Zoom transform uses `transformOrigin: 0 0` from a fit baseline, so zooming
  OUT below fit is pointless (shrinks toward the top-left corner) and zooming
  IN slides content toward a corner instead of anchoring. It feels broken.

## The fix (two tasks)

1. Make it a true fit-and-center canvas: the SVG renders at its intrinsic
   1280×720 size; the zoom transform does ALL fitting. `reset` and first mount
   compute a centered "fit" transform. Min scale drops below the fit scale so
   zoom-out actually pulls back; zoom-in anchors around the cursor/center.
2. Compact the sandbox controls into a left toolbar so the canvas gets much
   more vertical room (better aspect → zoom/pan feels natural). Guided mode's
   teaching layout (step text + big buttons) is unchanged.

## Global Constraints (apply to EVERY task; reviewers must enforce)

- Turkish UI only; plain short sentences. NO CSS `text-transform` anywhere
  (breaks Turkish casing).
- No new npm dependencies.
- `npm test`, `npx tsc -b`, `npm run lint`, `npm run build` all green after
  every task.
- Do NOT change `src/model.ts` semantics or any existing `tests/model.test.ts`
  assertion. This is a UI/interaction change only.
- Dark theme stays the visual default; do not change color tokens. Do not
  regress the command view (hover tooltip + "</> Komutlar" toggle) or the
  light/dark toggle.
- Keep `prefers-reduced-motion` handling intact.

## Task 1 — True fit-and-center pan/zoom canvas

**Files:** `src/useZoom.ts`, `src/Scene.tsx`, `src/App.tsx`, new `tests/zoom.test.ts`.

### 1a. Scene renders at intrinsic size (`src/Scene.tsx`)

- Export constants: `export const SCENE_W = 1280` and `export const SCENE_H = 720`.
- Change the root `<svg>` so the element itself is exactly 1280×720 CSS px
  (no letterbox double-fitting). Replace `className="h-full w-full"` with an
  explicit intrinsic size:
  ```tsx
  <svg
    viewBox={`0 0 ${SCENE_W} ${SCENE_H}`}
    width={SCENE_W}
    height={SCENE_H}
    className="block max-w-none"
    role="img"
    aria-label="GitHub görselleştirmesi"
  >
  ```
  `max-w-none` prevents any inherited `max-width:100%` from re-shrinking it.
  All existing scene contents/coordinates stay untouched.

### 1b. useZoom becomes fit-aware (`src/useZoom.ts`)

- Constants: `MIN_SCALE = 0.35`, `MAX_SCALE = 4`, `BUTTON_ZOOM_STEP = 1.25`,
  `WHEEL_SENSITIVITY = 0.0022` (keep last two).
- Keep `clampScale` and `zoomToward` exactly as they are (still exported).
- Add a pure, exported helper:
  ```ts
  // Centered fit: largest scale (× 0.94 breathing room) that fits the content
  // in the container, clamped, then translate so content is centered.
  export function fitTransform(
    containerW: number, containerH: number, contentW: number, contentH: number,
  ): ZoomTransform {
    const raw = Math.min(containerW / contentW, containerH / contentH) * 0.94
    const scale = clampScale(raw)
    return {
      scale,
      tx: (containerW - contentW * scale) / 2,
      ty: (containerH - contentH * scale) / 2,
    }
  }
  ```
- `useZoom(contentW: number, contentH: number)` now takes the content size.
- Add internal `fitToContainer()` that reads `containerRef.current` rect and
  sets the transform to `fitTransform(rect.width, rect.height, contentW, contentH)`.
- Track whether the user has manually adjusted the view since the last fit:
  a ref `adjustedRef` set to `true` on any wheel-zoom, drag-pan, or button
  zoom, and `false` inside `fitToContainer()`.
- `reset` → `setAnimate(true); fitToContainer()` (reset now re-frames both
  panels, not identity). Reset also clears `adjustedRef`.
- First mount + container resize: a `useEffect` that creates a
  `ResizeObserver` on `containerRef.current`, calls `fitToContainer()` once
  immediately (initial framing), and on every resize calls `fitToContainer()`
  ONLY if `adjustedRef.current === false` (don't yank a view the user has
  panned/zoomed). Disconnect on cleanup. Guard for missing ResizeObserver is
  not needed (evergreen browsers), but guard `containerRef.current` null.
- Initial `transform` state can stay `IDENTITY`; the mount effect immediately
  reframes before paint — use a `useLayoutEffect` for the initial fit so there
  is no visible identity flash. (ResizeObserver may live in a separate normal
  effect, or the same layout effect.)
- Wheel handler, `zoomAtCenter`/`zoomIn`/`zoomOut`, and the drag handlers keep
  their current math, but each of wheel-zoom, drag-move, and `zoomAtCenter`
  sets `adjustedRef.current = true`.

### 1c. App wires content size + non-letterboxed wrapper (`src/App.tsx`)

- `import { Scene, SCENE_W, SCENE_H } from './Scene'` and
  `const zoom = useZoom(SCENE_W, SCENE_H)`.
- The zoom-wrapper div must size to its content (the 1280×720 svg), NOT
  `h-full w-full`. Change its className from `zoom-wrapper h-full w-full …`
  to `zoom-wrapper w-max …` (or drop the sizing utilities entirely) so the
  transform scales the true-size svg. Keep the inline `transform` +
  `transformOrigin: '0 0'` exactly as now.
- The outer container div stays `h-full w-full … overflow-hidden` with the
  drag handlers.
- Give the three overlay zoom buttons `type="button"` (closes a v2.1 minor).

### 1d. Tests (`tests/zoom.test.ts`)

Pure-function tests only (no DOM):
- `clampScale`: clamps below MIN to 0.35, above MAX to 4, passes through 1.
- `zoomToward`: keeps the point under (cx,cy) fixed — e.g. from IDENTITY,
  `zoomToward({scale:1,tx:0,ty:0}, 100, 100, 2)` gives `{scale:2, tx:-100, ty:-100}`;
  the content point that was at (100,100) maps to (100,100) after transform.
- `fitTransform`:
  - wide container (1248×436, content 1280×720): scale = clamp(min(1248/1280,
    436/720)*0.94) = clamp(0.606*0.94=0.569) = 0.569; result is centered
    (tx = (1248-1280*0.569)/2, ty = (436-720*0.569)/2), both ≥ 0.
  - square-ish container where width binds; assert scale uses the width ratio.
  - tiny container that forces clamp to MIN_SCALE (0.35).

### Manual verification (controller, in-browser)

- On load, both panels are centered and framed with modest margins.
- Ctrl/⌘+scroll zooms toward the cursor; both panels grow/shrink together.
- Zoom-out (− button, or scroll) now pulls the whole scene back smaller —
  visibly useful, not a corner shrink.
- Drag pans; the GitHub panel and computer panel move together as one canvas.
- ⤢ / double-click reframes both panels centered.
- Resizing the window (or switching guided↔sandbox) reframes when untouched.

## Task 2 — Compact sandbox controls (left toolbar), taller canvas

**Files:** `src/App.tsx` (+ `src/index.css` only if a helper class is needed).
Depends on Task 1 (uses the fit canvas; the taller container it creates makes
the fit framing look right).

### Layout

- Guided mode: UNCHANGED. Canvas on top, teaching footer (event line, step
  text, cheat sheet, big centered buttons, keyboard hint) below.
- Sandbox mode: restructure so the action controls live in a **left vertical
  toolbar** beside the canvas, and the bulky bottom footer is gone in sandbox.
  - `<main>` in sandbox becomes a horizontal flex: a fixed-width left rail
    (`w-60 shrink-0`, vertically scrollable `overflow-y-auto`) + the canvas
    container (`flex-1`). Keep the existing canvas container (drag handlers,
    overlay, rounded panel) as the flex-1 child.
  - Rail contents, top to bottom:
    - A one-line status: the `EventLine` (reuse the component) so the last
      action ("main push'landı…") is still visible.
    - A short hint line: "Serbestsin: boz, kaydet, geri dön." (smaller,
      `text-[var(--ink-muted)]`).
    - The sandbox action buttons as a vertical stack, each `w-full`,
      left-aligned text, compact (`!px-3 !py-2 !text-sm`, keep `rounded-2xl`
      → use `rounded-xl` for the tighter look). Keep every button's `kind`
      color, `disabled`, `data-cmd`, and the `showCmds` inline command line
      exactly as today (command view must keep working in the rail).
- Do NOT change the sandbox action set, ordering, labels, cmds, or handlers
  in `sandboxActions()` — only where/how they render.
- The zoom overlay stays bottom-right inside the canvas container.

### Result / rationale

Removing the wide bottom bar gives the canvas the full height between header
and window bottom. Container aspect moves from ~2.86 toward ~1.5, so the
fit framing from Task 1 shows both panels larger with small side margins —
directly answering "panels feel small / zoom feels pointless."

### Manual verification (controller, in-browser)

- Sandbox: left rail holds all actions compactly; canvas fills the rest and
  is noticeably taller than before; both panels framed larger at default fit.
- "</> Komutlar" toggle still shows each rail button's git command inline;
  hover tooltip still works.
- Guided mode still looks/behaves exactly as before (unchanged footer).
- Light and dark themes both look clean in the new rail (tokens only).
- Buttons wrap/scroll gracefully if the rail is taller than the window.
