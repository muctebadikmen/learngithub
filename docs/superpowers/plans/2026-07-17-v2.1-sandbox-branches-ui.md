# v2.1 — Sandbox multi-branch, command view, zoom, themes

Owner feedback (2026-07-17): the sandbox is the best part but (1) only one
branch can exist at a time, which teaches branching badly — multiple branches
must coexist, each visibly carrying different work, switchable; (2) how branch
work gets pushed must be modeled correctly; (3) every action button must be
able to show the real git command (hover tooltip + a global toggle);
(4) UI polish: Ctrl/Cmd+scroll zoom on the scene, pan, and a beautiful light
theme alongside dark.

## Global Constraints

- Turkish UI only. Plain, short, direct sentences — no story fluff. Real git
  terms stay English with Turkish gloss where space allows.
- NO CSS `text-transform` anywhere (breaks Turkish casing).
- `src/model.ts` remains a small display-state driving one SVG picture — it
  must NOT grow toward a git engine. No files, no staging, no index.
- After EVERY task: `npm test` green, `npx tsc -b` clean, `npm run lint`
  clean, `npm run build` succeeds.
- Keep existing file layout: `src/model.ts`, `src/chapters.ts`,
  `src/Scene.tsx`, `src/App.tsx`, `src/index.css`, tests in
  `tests/model.test.ts`. Task 3 may add `src/useZoom.ts`. No new npm
  dependencies.
- Animations stay ~600ms via the existing `.scene-node` CSS classes;
  `prefers-reduced-motion` must keep disabling them.
- Scene SVG viewBox stays `0 0 1280 720`.
- The guided 7-chapter flow must keep working unchanged from the player's
  point of view (single branch at a time there).
- Existing UI conventions: buttons built from the `BTN` map in App.tsx;
  sandbox buttons come from `sandboxActions(s)`; `lastEvent` strings live in
  model actions.

## Task 1 — Multi-branch model + mechanical UI adaptation

**Goal:** up to 3 side branches may exist simultaneously, each with its own
work; switching, merging, deleting, pushing and PRs operate on the *current*
branch. Scene draws every branch on its own lane. Sandbox exposes it.

### model.ts changes (exact)

Types:

```ts
export type Branch = { name: string; forkId: string; laneIdx: number } // laneIdx ∈ 1..3
```

`ModelState`: REMOVE `branchName`, `branchFromId`, `currentLane`, and the
`Lane` type. ADD:
- `branches: Branch[]` — active side branches (max 3)
- `currentBranch: string` — `'main'` or an active branch name

`Commit`: replace `lane: Lane` with:
- `branch: string` — `'main'` or the branch name it was created on
- `laneIdx: number` — 0 for main, else the creating branch's `laneIdx`;
  stored forever so history renders on its lane even after the branch is
  merged/deleted and the lane number gets reused.

Helpers (exported):
- `tip(s, name)` — last commit with `branch === name`; for an active branch
  with no commits yet, its fork commit; `null` when neither exists.
- `activeBranch(s)` — the `Branch` for `currentBranch`, or null on main.
- `branchCommits(s, name)` — commits with `branch === name` AND
  `x > fork.x` (fork = that branch's `forkId` commit). For `'main'` return
  all `branch === 'main'` commits.
- `abandonedIds(s)` — unchanged idea, but walk from `tip(s, head.branch)`.

Actions (all pure, all set a Turkish `lastEvent`):
- `createRepo` — unchanged, first commit gets `branch: 'main', laneIdx: 0`.
- `createBranch(s, name?)` — guards: `hasRepo`, `currentBranch === 'main'`,
  `branches.length < 3`, `headId` non-null, name not already active and not
  `'main'`. Default name: first unused of `['deneme', 'tasarim', 'ozellik']`.
  `laneIdx` = smallest unused value in 1..3. `forkId = headId`.
  Sets `currentBranch = name`. Clears a leftover `pr` only if
  `pr.status === 'merged'`. Event: `“{name}” branch'i açıldı — main artık güvende.`
- `switchBranch(s, name)` — guard: name is `'main'` or active. Head becomes
  `tip(s, name)`; `workLook` copied from it; `dirty = false` (uncommitted
  work is discarded — if it was dirty, append ` (kaydedilmemiş değişiklik atıldı)`
  to the event). Event: `“{name}” hattına geçildi.`
- `commit` — unchanged, stamps `branch: currentBranch` and the right `laneIdx`.
- `goBack` — unchanged semantics on the current branch (dirty → discard;
  else step to parent; crossing the fork stays blocked unless `head.isMerge`).
- `mergeBranch(s)` — merges the CURRENT branch into main. Guards:
  `currentBranch !== 'main'` and `branchCommits(s, currentBranch).length > 0`.
  Merge commit on main (`branch:'main'`, `laneIdx:0`,
  `mergeFromId = branch tip id`, look = branch tip look). Remove the branch
  from `branches`; `currentBranch = 'main'`. If `pr?.from === merged name`,
  set its status `'merged'`, else leave `pr` alone.
- `deleteBranch(s)` — deletes the CURRENT branch. Guard
  `currentBranch !== 'main'`. Remove its `branchCommits` from `commits` and
  `pushedIds`, remove it from `branches`, `currentBranch = 'main'`, head =
  `tip(s,'main')`, clear a `pr` whose `from` is this branch.
- `push(s)` — pushes the CURRENT branch like real git: the reachable set of
  the current head (walk `parentId` chain AND, through merge commits,
  `mergeFromId` chains), minus `abandonedIds`. UNION into `pushedIds`
  (pushing one branch must not unpush another). No-op (return `s`) when
  nothing new. Event: `“{currentBranch}” push'landı — {n} yeni commit GitHub'da.`
- `openPR(s)` — guards: on a branch, `branchCommits` non-empty, **all of that
  branch's commits are already pushed** (this is the teaching: push, THEN
  PR), no existing non-merged `pr`. `pr = { from: currentBranch, status: 'open' }`.
- `approvePR` — unchanged.
- `laptopDie` — unchanged. `restoreFromCloud(s)` — keep pushed commits;
  keep only branches that still have ≥1 pushed commit; `currentBranch = 'main'`;
  head = last kept main commit (or the empty-repo reset as today).
- `initialModel` — `branches: []`, `currentBranch: 'main'`.

## Task 2 — Command view (hover + global toggle)

**Goal:** every action can reveal the real git command.

- New `cmd` field on guided `StepButton`s and sandbox buttons (a string).
  Exact commands:
  - Repo oluştur → `git init`
  - Commit at → `git add . && git commit -m "mesaj"`
  - Geri dön → dynamic: dirty state → `git restore .` ; else → `git reset --hard HEAD~1`
  - Push'la → `git push -u origin {currentBranch}` (guided push steps: `git push -u origin main`)
  - Branch aç → `git switch -c {isim}` (guided: the actual name used)
  - geç: {name} → `git switch {name}`
  - Merge et → `git switch main && git merge {isim}`
  - Sil / branch'i sil → `git branch -D {isim}`
  - PR aç → `gh pr create`
  - PR'ı onayla → `GitHub arayüzünde: Review → Approve`
  - GitHub'dan geri indir → `git clone {repo-adresi}`
  - Bilgisayarı çökert, Eyvah, Olamaz, Devam, Sıfırla, Sandbox'ta oyna →
    no command (omit `cmd`).
- **Hover tooltip:** any action button WITH a `cmd` shows it on hover in a
  small monospace dark tooltip above the button. Pure CSS: button wrapper
  carries `data-cmd`, tooltip via `::after`/`::before` in index.css
  (`position:absolute`, no JS). Must not fire for buttons without `cmd`,
  must not clip at screen edges for the row's outermost buttons
  (`max-width` + centered transform is fine).
- **Global toggle:** header button `</> Komutlar` (ghost style, both modes).
  When ON, every action button with a `cmd` renders the command as a second
  line inside the button: monospace, ~12px, reduced opacity, under the
  Turkish label. State lives in App (`useState`), OFF by default, no
  persistence.
- The toggle and tooltips work in BOTH guided and sandbox modes.
- Add nothing else — no command palette, no copy buttons.

**Manual check:** toggle on → labels show commands; hover shows tooltip;
"Geri dön" command changes between `git restore .` (dirty) and
`git reset --hard HEAD~1` (clean).

## Task 3 — Scene zoom & pan

**Goal:** Ctrl/Cmd + scroll zooms the scene, drag pans it, easy reset.

- New `src/useZoom.ts` hook owning `{ scale, tx, ty }` + handlers; used by
  App around the `<Scene>` container.
- Wheel with `ctrlKey || metaKey` (covers trackpad pinch): `preventDefault`,
  multiply scale by `exp(-deltaY * 0.0022)`, clamp scale to [0.6, 3],
  zoom **toward the cursor** (keep the point under the cursor fixed:
  standard `tx' = cx - (cx - tx) * k` math using the container's
  `getBoundingClientRect`). Plain scroll without the modifier must remain
  untouched (page has no scroll anyway, but do not preventDefault it).
- Pointer drag on the scene container pans (`setPointerCapture`,
  `cursor: grab`/`grabbing`). Buttons/footer are outside the container and
  unaffected.
- Apply as CSS `transform: translate(tx, ty) scale(s)` on an inner wrapper
  div, `transform-origin: 0 0`. No transition during wheel/drag; 150ms ease
  transition when using the buttons/reset.
- Overlay controls, bottom-right INSIDE the scene area: `−`, `+`, `⤢`
  (reset to scale 1, tx=ty=0). Double-click on the scene also resets.
  Buttons zoom around the container center in steps of ×1.25.
- Tiny hint under the controls: `Ctrl/⌘ + scroll` (muted, 11px).
- Wheel listener must be non-passive (attach via `ref` +
  `addEventListener(..., { passive: false })`, not React's `onWheel`).

**Manual check:** ctrl+scroll zooms toward cursor, drag pans, dblclick and ⤢
reset, buttons still clickable, guided flow unaffected.

## Task 4 — Light/dark themes + visual polish

**Goal:** first-class light theme, dark stays default; overall finish.

- All colors become CSS custom properties in `index.css`:
  `:root` (dark values) and `:root[data-theme="light"]` overrides. Tokens at
  minimum: `--bg`, `--panel`, `--panel-2` (GitHub panel), `--panel-stroke`,
  `--ink`, `--ink-muted`, `--ink-faint`, `--card-bg`, `--card-stroke`,
  `--node-ring` (bg-colored node border), `--pill-bg`, `--chip-stroke`,
  `--danger`, `--warn`, `--accent`, `--accent-ink`,
  `--lane-0` … `--lane-3`, `--tooltip-bg`, `--tooltip-ink`.
- Scene.tsx and App.tsx replace every hex literal with `var(--…)` (SVG
  `fill`/`stroke` accept `var()`; Tailwind arbitrary values like
  `bg-[var(--bg)]`, `text-[var(--ink)]` for the shell). The `LANE_COLORS`
  array becomes `['var(--lane-0)', …]`.
- Dark values = the current palette (no visual change in dark mode).
- Light palette (must look genuinely good, not inverted-dark):
  `--bg #eef0f7`; panels `#ffffff` with stroke `#e3e5ef` and a soft shadow
  (`box-shadow: 0 10px 32px rgb(15 23 42 / 0.07)` on the shell panels via a
  class — for the SVG panels use a slightly darker stroke instead);
  GitHub panel `#f6f8fa`; `--ink #171a24`, `--ink-muted #5b6070`,
  `--ink-faint #9aa0b0`; lanes darkened for contrast:
  `--lane-0 #059669`, `--lane-1 #7c3aed`, `--lane-2 #d97706`,
  `--lane-3 #0284c7`; `--danger #dc2626`, `--warn #b45309`;
  app-card `#ffffff` stroke `#d9dbe6`; event line uses `--warn`.
  Buttons keep emerald primary with white text adjustments as needed for
  AA contrast.
- Header toggle `🌙/☀️` sets `document.documentElement.dataset.theme`,
  persists to `localStorage['rehber-theme']`, initial value: stored value,
  else `'dark'` (dark stays the default for the owner's video). Set CSS
  `color-scheme` accordingly.
- Polish pass (small, targeted): buttons get a subtle hover lift
  (`translateY(-1px)` + shadow) and consistent focus-visible rings; the
  event line and step text keep their animations; header gets a hairline
  bottom border (`--panel-stroke`); nothing else.
- `prefers-reduced-motion` continues to disable all animations incl. the
  new lift.

**Manual check:** toggle themes in guided AND sandbox; every scene element
(panels, nodes, pills, PR card, tooltips, skull overlay) must be legible in
light mode; reload keeps the chosen theme; dark mode is pixel-equivalent to
before.
