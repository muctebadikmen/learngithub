# v3.1 — Sahne büyümesi + anlamlı zoom (design)

**Tarih:** 2026-07-17
**Branch:** `v3.1-scene-grow-zoom`

## Problem

Commit / branch sayısı arttıkça timeline noktaları sabit genişlikli sahnenin içine
**sıkıştırılıyor**; noktalar arası boşluk daralıyor, etiketler üst üste biniyor
(`Değişiklik…DeğişiklikDeğişiklik`). GitHub paneli daha dar olduğu için orada çakışma
en kötü seviyede.

Kök neden `Scene.tsx > timelineSlots`:

```ts
const gap = maxX > 0 ? Math.min(120, (endX - startX) / maxX) : 0
```

Boşluk commit sayısıyla ters orantılı → sıkışma. **Zoom bunu çözemez:** zoom sadece
bu sıkışmış sabit resmi ölçekler; yakınlaştırınca hem boşluk hem etiket birlikte
büyür, üst üste binme oranı sabit kalır. `MIN_SCALE = 0.5` olduğu için "uzaklaştır"
da erken tabana çarpıp işe yaramıyor. Kutular (`Senin bilgisayarın` / `GitHub`) sabit
genişlikli olduğundan içeriğe göre büyümüyor.

## Çözüm (onaylanan yön)

Sıkıştırmayı kaldır; sahne içeriğe göre **yatayda büyüsün**; zoom o büyük sahnede
gezinme aracı olsun.

1. **Sabit boşluk.** `timelineSlots` sabit bir `GAP` (≈112px) kullanır; commit sayısı
   ne olursa olsun boşluk daralmaz → etiketler asla çakışmaz. Hem yerel (bilgisayar)
   hem bulut (GitHub) timeline'ı için.

2. **İçeriğe göre büyüyen sahne.** Yeni saf yardımcı `sceneLayout(state)` commit
   sayılarından şunları hesaplar: bilgisayar panelinin genişliği, GitHub panelinin
   yeni x konumu ve genişliği, toplam `sceneW`. Yükseklik sabit (720) — kalabalık
   yalnızca yatayda oluşuyor.

3. **GitHub tarafı tek `translate` grubunda.** GitHub paneli rect'i + içeriği tek bir
   `<g transform="translate(dx,0)">` içine alınır; `dx` panel yeni konumuna göre
   hesaplanır. Böylece GitHub'ın 800-tabanlı onlarca iç koordinatı elden geçirilmeden
   sağa kayar (kod karmaşası yok).

4. **Anlamlı zoom (`useZoom.ts`).**
   - `MIN_SCALE` 0.5 → **0.12**: "uzaklaştır" artık uzun geçmişin tamamını çerçeveye
     sığdırabilir.
   - **Akıllı çerçeveleme:** kullanıcı elle zoom/pan yapmadıysa (`adjustedRef` false),
     sahne büyüdükçe görünüm otomatik yeniden çerçevelenir (her şey görünür kalır);
     elle oynanmışsa dokunulmaz, `⤢` (reset) sıfırlar.

5. **`App.tsx`.** `useZoom`'a sabit `SCENE_W` yerine `sceneLayout(model).sceneW` verilir.
   `Scene` kendi layout'unu `state`'ten saf olarak yeniden hesaplar (prop threading yok).

## Layout matematiği (referans)

`GAP = 112`, panel-arası boşluk `32`, sağ/etiket payları aşağıdaki gibi. Boş/az-commit
durumunda **birebir mevcut geometriye** (1280×720, paneller 24/800) düşer.

```
maxLocalX          = state.commits.length ? max(c.x) : 0
computerContentR   = 116 + maxLocalX*GAP + 72          // 72 = etiket/📍pin payı
computerPanelR     = max(768, computerContentR)         // 768 = mevcut sağ kenar
computerPanelW     = computerPanelR - 24
repoBoxW           = computerPanelW - 48

maxCloudX          = cloudCommits.length ? max(c.x) : 0 // cloud = pushed + remoteExtra
ghContentR_local   = max(1256, 852 + maxCloudX*GAP + 60)
ghPanelW           = ghContentR_local - 800
ghPanelX           = computerPanelR + 32                 // yeni sol kenar
dx                 = ghPanelX - 800                      // translate grubu

sceneW             = ghPanelX + ghPanelW + 24
sceneH             = 720 (sabit)
```

Sıfır-commit teyidi: `computerPanelR=768`, `dx=0`, `ghPanelW=456`, `sceneW=1280`
→ mevcut sahneyle özdeş (regresyon yok).
